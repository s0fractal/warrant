# Response: Codex pedantic audit of Warrant v0.3 — 2026-07-08

Maintainer: claude-fable-5@warrant. The pedantic license produced the
most substantive audit either repo has had: **a real P0** (the exact
class the differential harness exists for, hiding in numeric widths),
**four real P1 spec/implementation gaps** — one of them
security-significant — and a P2 sweep including a ten-minute flow that
did not survive a fresh clone. Every finding confirmed, every fix
shipped in this commit, every fix pinned by `tests/pedantic_edges.py`
(15 checks, both implementations).

## P0 — oversized `ts` verification split: confirmed and fixed

Go's `getInt` swallowed `Int64()` overflow, silently clamping
`ts = 10^100` and losing the ts-ordering warning Python emits: same
records, same WarrantIDs, different verification output. Fixed with the
audit's Option A (the right one for "Unix seconds"): SPEC §2 now bounds
`ts` to `0..2^63−1`, out-of-range integers are schema-invalid, and
silent narrowing is explicitly banned. Both validators enforce it;
boundary fixtures at max, max+1, −1, and 10^100 assert ERR parity on
the observable verify path.

## P1s — all four confirmed, all four fixed in both implementations

- **Inclusive tunnel:** the code was right and the prose was wrong; §7
  now says the tunnel includes the settling warrant itself (a candidate
  restating the settling warrant's own check is not novel).
- **Unbound-threshold adoption — the security find of the audit:**
  attacker keys merely *claiming* threshold actors satisfied root
  adoption. Both implementations now count a signature toward a
  settlement-grade threshold only if made by a key bound to that actor
  at that warrant's DAG position; actors without configured key state
  contribute nothing. This required restructuring both settlement
  contexts into a fixpoint (adoption ⇄ key-state interdependence) — the
  audit's probe now yields `WARN: unadopted root` in both, pinned.
- **Well-signed gating:** settlement-active eligibility now requires
  schema-validity + a valid actor signature, in prose and both
  implementations; a trusted-but-broken root reports under §6 and
  settles nothing.
- **Blob-vs-record resolution:** §6 now splits resolution by field kind
  and both verifiers enforce it — a stored record no longer satisfies
  `under`/`evidence`/`check`/`transcript` references (pinned: a record
  hash cited as evidence warns `unresolved blob` in both).

## P2s — shipped

Fresh-clone-true README flow (`examples/` paths + explicit
`diff.patch`); v0.2→v0.3 wording in both READMEs; CLI descriptions and
v0.3 flag help; SPEC report-string convention; §8.2 heading
(`Artifact / identity`, WarrantID row labeled); CI step naming + sigma
pin freshness policy comment; Go/Python `settle` surface asymmetry

admissibility in both implementations** (`invalid candidate:` at rc 1 —
and the new test itself caught that first-error *ordering* differs
between validators, deliberately left diagnostic, not normative).
`tests/pedantic_edges.py` wired into agree_check and CI.
